**How can I ensure that my code, program, or software is functional and secure?**
I ensure that my code, program, or software is functional and secure by defining its requirements and testing them thoroughly. Defining requirements means creating detailed specifications of exactly what I want the program to do; for example, a requirement could be _"Age variable must allow for a range between 0 and 122 inclusive."_ To test requirements, testing methods must be applied, such as unit, static, dynamic, and compatibility testing. 

**How do I interpret user needs and incorporate them into a program?**
I interpret user needs by determining which user needs appear to be the most desired and popular. Then, I incorporate them into a program by using them as a basis to build requirements on.

**How do I approach designing software?**
I approach designing software by considering how I can create the software in a way that conforms to its requirements. For example, if one of the software requirements is _"Description must be at least 50 characters long,"_ then I would ensure that the software design includes a Description variable and a constructor that sets a minimum size for the Description variable.
